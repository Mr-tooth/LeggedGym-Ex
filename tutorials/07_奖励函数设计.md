# 教程七：奖励函数设计

## 7.1 奖励函数概述

奖励函数是强化学习的核心，它告诉智能体什么是"好"的行为。LeggedGym-Ex 提供了丰富的奖励函数库，并采用**奖励项组合**的方式设计总奖励。

## 7.2 奖励系统架构

### 奖励计算流程

```python
def compute_reward(self):
    self.rew_buf[:] = 0.
    for i in range(len(self.reward_functions)):
        name = self.reward_names[i]
        rew = self.reward_functions[i]() * self.reward_scales[name]
        self.rew_buf += rew
        self.episode_sums[name] += rew
```

每个奖励项被单独计算，然后乘以对应的权重（scale），最后相加得到总奖励。

### 奖励函数准备

在初始化时，系统会自动收集所有非零权重的奖励项：

```python
def _prepare_reward_function(self):
    self.reward_functions = []
    self.reward_names = []
    self.reward_scales = class_to_dict(self.cfg.rewards.scales)

    for name, scale in self.reward_scales.items():
        if name == "termination":
            continue
        if scale != 0:
            self.reward_names.append(name)
            self.reward_functions.append(getattr(self, f"_reward_{name}"))
```

## 7.3 可用奖励函数详解

### 7.3.1 命令跟踪奖励

#### `_reward_tracking_lin_vel` - 线速度跟踪

```python
def _reward_tracking_lin_vel(self):
    lin_vel_error = torch.sum(torch.square(
        self.commands[:, :2] - self.simulator.base_lin_vel[:, :2]), dim=1)
    return torch.exp(-lin_vel_error / 0.25)
```

**用途**：奖励机器人按命令的线速度移动
**参数**：无
**返回**：指数衰减的奖励，误差越小奖励越高

#### `_reward_tracking_ang_vel` - 角速度跟踪

```python
def _reward_tracking_ang_vel(self):
    ang_vel_error = torch.square(
        self.commands[:, 2] - self.simulator.base_ang_vel[:, 2])
    return torch.exp(-ang_vel_error / 0.25)
```

**用途**：奖励机器人按命令的角速度转向

### 7.3.2 平滑性奖励

#### `_reward_lin_vel_z` - 垂直速度惩罚

```python
def _reward_lin_vel_z(self):
    return torch.square(self.simulator.base_lin_vel[:, 2])
```

**用途**：惩罚垂直方向的抖动
**权重建议**：负值（如 -0.5）

#### `_reward_ang_vel_xy` - 俯仰/滚转角速度惩罚

```python
def _reward_ang_vel_xy(self):
    return torch.sum(torch.square(self.simulator.base_ang_vel[:, :2]), dim=1)
```

**用途**：惩罚不必要的俯仰和翻滚

#### `_reward_orientation` - 姿态惩罚

```python
def _reward_orientation(self):
    return torch.sum(torch.square(self.simulator.projected_gravity[:, :2]), dim=1)
```

**用途**：惩罚身体倾斜

#### `_reward_dof_vel` - 关节速度惩罚

```python
def _reward_dof_vel(self):
    return torch.sum(torch.square(self.simulator.dof_vel), dim=1)
```

**用途**：惩罚关节运动过快

#### `_reward_dof_acc` - 关节加速度惩罚

```python
def _reward_dof_acc(self):
    return torch.sum(torch.square(
        (self.simulator.dof_vel - self.last_dof_vel) / self.dt), dim=1)
```

**用途**：惩罚关节加速度，使运动更平滑

#### `_reward_action_rate` - 动作变化率惩罚

```python
def _reward_action_rate(self):
    return torch.sum(torch.square(self.actions - self.last_actions), dim=1)
```

**用途**：惩罚动作变化过快

#### `_reward_action_smoothness` - 动作平滑性惩罚

```python
def _reward_action_smoothness(self):
    return torch.sum(torch.square(
        self.actions - 2 * self.last_actions + self.llast_actions), dim=1)
```

**用途**：惩罚动作的二阶变化，使运动更平滑

#### `_reward_torques` - 力矩惩罚

```python
def _reward_torques(self):
    return torch.sum(torch.square(self.simulator.torques), dim=1)
```

**用途**：惩罚力矩消耗，节省能量

### 7.3.3 限制惩罚

#### `_reward_dof_pos_limits` - 关节位置限制惩罚

```python
def _reward_dof_pos_limits(self):
    out_of_limits = -(self.simulator.dof_pos - self.dof_pos_limits[:, 0]).clip(max=0.)
    out_of_limits += (self.simulator.dof_pos - self.dof_pos_limits[:, 1]).clip(min=0.)
    return torch.sum(out_of_limits, dim=1)
```

**用途**：惩罚关节超出位置限制

#### `_reward_collision` - 碰撞惩罚

```python
def _reward_collision(self):
    return torch.sum(
        torch.norm(self.simulator.link_contact_forces[:, self.simulator.penalize_contacts_on, :], dim=-1), dim=1)
```

**用途**：惩罚不期望的碰撞

#### `_reward_torque_limits` - 力矩限制惩罚

```python
def _reward_torque_limits(self):
    out_of_limits = torch.abs(self.simulator.torques) - self.torque_limits
    return torch.sum(out_of_limits.clip(min=0.), dim=1)
```

**用途**：惩罚力矩超出安全限制

### 7.3.4 步态奖励

#### `_reward_feet_air_time` - 脚离地时间奖励

```python
def _reward_feet_air_time(self):
    contact = self.simulator.contact_forces[:, self.simulator.feet_indices, 2] > 1.
    contact_filt = torch.logical_or(contact, self.last_contacts)
    self.last_contacts = contact
    first_contact = (self.feet_air_time > 0.) * contact_filt
    self.feet_air_time += self.dt
    rew_air_time = torch.sum((self.feet_air_time - 0.5) * first_contact, dim=1)
    self.feet_air_time *= ~contact_filt
    return rew_air_time
```

**用途**：奖励脚有适当的离地时间，促进步态形成

#### `_reward_foot_clearance` - 脚间隙奖励

```python
def _reward_foot_clearance(self):
    foot_pos = self.simulator.feet_pos[:, :, 2] - self.cfg.rewards.foot_height_offset
    error = foot_pos - self.cfg.rewards.foot_clearance_target
    rew = torch.exp(-torch.square(error) /
                    self.cfg.rewards.foot_clearance_tracking_sigma ** 2)
    return torch.sum(rew * (~self.feet_contact), dim=-1)
```

**用途**：奖励脚保持适当的离地高度

### 7.3.5 基础高度奖励

#### `_reward_base_height` - 基座高度惩罚

```python
def _reward_base_height(self):
    return torch.square(self.simulator.base_pos[:, 2] - self.cfg.rewards.base_height_target)
```

**用途**：惩罚基座高度偏离目标值

### 7.3.6 终止奖励

#### `_reward_termination` - 终止惩罚

```python
def _reward_termination(self):
    return self.reset_buf.float()
```

**用途**：惩罚提前终止（注意：此奖励单独处理，不在循环中）

## 7.4 配置奖励函数

### 奖励配置示例

```python
class rewards(LeggedRobotCfg.rewards):
    soft_dof_pos_limit = 0.9
    base_height_target = 0.36
    only_positive_rewards = True  # 裁剪掉负奖励

    class scales(LeggedRobotCfg.rewards.scales):
        # 命令跟踪（正奖励）
        tracking_lin_vel = 1.0
        tracking_ang_vel = 0.5

        # 平滑性（负奖励 = 惩罚）
        lin_vel_z = -0.5
        ang_vel_xy = -0.05
        orientation = -1.0
        dof_vel = -5.e-4
        dof_acc = -2.e-7
        action_rate = -0.01
        action_smoothness = -0.01
        torques = -2.e-4

        # 限制（负奖励）
        dof_pos_limits = -1.0
        collision = -1.0

        # 步态（正奖励）
        feet_air_time = 1.0
        foot_clearance = 0.5
```

### 关键配置参数

| 参数 | 说明 |
|-----|------|
| `soft_dof_pos_limit` | 关节软限制比例，在此比例内开始惩罚 |
| `base_height_target` | 目标基座高度 |
| `foot_clearance_target` | 目标脚离地高度 |
| `only_positive_rewards` | 是否只保留正奖励 |

## 7.5 奖励设计最佳实践

### 7.5.1 奖励权重调优流程

1. **从简单开始**
   ```python
   # 初始配置：只保留最基本的奖励
   class scales:
       tracking_lin_vel = 1.0
       tracking_ang_vel = 0.5
       collision = -1.0
       dof_pos_limits = -1.0
   ```

2. **添加平滑性奖励**
   ```python
   class scales:
       tracking_lin_vel = 1.0
       tracking_ang_vel = 0.5
       dof_vel = -1.e-4  # 添加
       action_rate = -0.01  # 添加
       collision = -1.0
       dof_pos_limits = -1.0
   ```

3. **添加步态奖励**
   ```python
   class scales:
       tracking_lin_vel = 1.0
       tracking_ang_vel = 0.5
       dof_vel = -1.e-4
       action_rate = -0.01
       feet_air_time = 1.0  # 添加
       foot_clearance = 0.5  # 添加
       collision = -1.0
       dof_pos_limits = -1.0
   ```

### 7.5.2 常见问题调试

#### 问题 1：机器人移动但步态奇怪

**可能原因**：缺少步态奖励或权重太小

**解决方法**：
```python
feet_air_time = 2.0  # 增加
foot_clearance = 1.0  # 增加
```

#### 问题 2：机器人抖动严重

**可能原因**：平滑性惩罚不够

**解决方法**：
```python
dof_vel = -5.e-4  # 增加绝对值
dof_acc = -2.e-7  # 增加绝对值
action_rate = -0.05  # 增加绝对值
```

#### 问题 3：机器人不跟踪命令

**可能原因**：跟踪奖励权重太小，或其他惩罚过大

**解决方法**：
```python
tracking_lin_vel = 2.0  # 增加
tracking_ang_vel = 1.0  # 增加

# 同时检查是否有过大的惩罚
lin_vel_z = -0.2  # 减小绝对值
```

### 7.5.3 奖励设计原则

1. **稀疏 vs 密集**
   - 初期使用密集奖励帮助学习
   - 后期可考虑稀疏奖励提高鲁棒性

2. **正奖励 vs 负奖励**
   - 正奖励：鼓励期望行为
   - 负奖励：惩罚不期望行为
   - 建议组合使用

3. **奖励量级**
   - 保持总奖励在合理范围（如 0-10/回合）
   - 避免某个奖励项主导

4. **奖励塑形**
   - 使用指数函数奖励小误差
   - 使用二次函数惩罚大误差

## 7.6 自定义奖励函数

### 添加新奖励项

假设我们想添加一个奖励，鼓励机器人的脚与地面接触时保持低速度：

```python
class MyRobot(LeggedRobot):
    # ... 其他方法 ...

    def _reward_foot_landing_vel(self):
        """奖励脚着地时速度低"""
        # 获取脚的垂直速度
        foot_vel_z = self.simulator.feet_vel[:, :, 2]

        # 只在接触时惩罚
        contact = self.simulator.contact_forces[:, self.simulator.feet_indices, 2] > 1.

        # 速度平方乘以接触掩码
        reward = -torch.sum(torch.square(foot_vel_z) * contact, dim=1)

        return reward
```

然后在配置中启用：

```python
class scales(LeggedRobotCfg.rewards.scales):
    # ... 其他奖励 ...
    foot_landing_vel = 0.5  # 添加这一行
```
