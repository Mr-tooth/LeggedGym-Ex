# 教程四：配置系统详解

## 4.1 配置系统概述

LeggedGym-Ex 使用**类继承**的方式组织配置，具有以下特点：

- 分层配置：基础配置 → 机器人通用配置 → 具体任务配置
- 环境配置与训练配置分离
- 支持运行时参数覆盖

## 4.2 配置类的结构

### 两种配置类型

每个任务都有两个配置类：

1. **环境配置** (`*Cfg`)：定义环境、机器人、奖励等参数
2. **训练配置** (`*CfgPPO`)：定义算法、优化器、日志等参数

### 示例：Go2 配置

```python
from legged_gym.envs.base.legged_robot_config import LeggedRobotCfg, LeggedRobotCfgPPO

class GO2Cfg(LeggedRobotCfg):
    """Go2 环境配置"""

    class env(LeggedRobotCfg.env):
        num_envs = 4096
        num_observations = 45
        num_actions = 12

    class rewards(LeggedRobotCfg.rewards):
        soft_dof_pos_limit = 0.9
        base_height_target = 0.36

        class scales(LeggedRobotCfg.rewards.scales):
            tracking_lin_vel = 1.0
            collision = -1.0

class GO2CfgPPO(LeggedRobotCfgPPO):
    """Go2 训练配置"""

    class runner(LeggedRobotCfgPPO.runner):
        run_name = 'simple_rl'
        experiment_name = 'go2'
        max_iterations = 1500
```

## 4.3 环境配置详解

### 主要配置项

`LeggedRobotCfg` 包含以下主要配置类：

| 配置类 | 说明 |
|-------|------|
| `env` | 环境基本参数 |
| `terrain` | 地形设置 |
| `init_state` | 初始状态 |
| `control` | 控制器参数 |
| `asset` | 机器人资产 |
| `rewards` | 奖励设置 |
| `commands` | 命令生成 |
| `domain_rand` | 领域随机化 |
| `normalization` | 观测归一化 |
| `sim` | 模拟器参数 |

### env - 环境基本参数

```python
class env:
    num_envs = 4096           # 并行环境数量
    num_observations = 45     # 观测维度
    num_privileged_obs = None # 特权观测维度
    num_actions = 12          # 动作维度
    episode_length_s = 20     # 回合长度（秒）
```

### terrain - 地形设置

```python
class terrain:
    mesh_type = 'plane'        # 地形类型: plane, heightfield, trimesh
    curriculum = True          # 是否使用课程学习
    terrain_length = 6.0       # 地形块长度
    terrain_width = 6.0        # 地形块宽度

    # 地形类型比例: [平滑, 粗糙, 上楼梯, 下楼梯, 离散]
    terrain_proportions = [0.1, 0.1, 0.35, 0.25, 0.2]
```

### init_state - 初始状态

```python
class init_state:
    pos = [0.0, 0.0, 0.36]    # 初始位置 [x, y, z]
    rot = [0.0, 0.0, 0.0, 1.0] # 初始姿态四元数 [x, y, z, w]
    lin_vel = [0.0, 0.0, 0.0] # 初始线速度
    ang_vel = [0.0, 0.0, 0.0] # 初始角速度
    default_joint_angles = {   # 默认关节角度
        "FL_hip_joint": 0.1,
        "FR_hip_joint": 0.1,
        # ...
    }
```

### control - 控制器参数

```python
class control:
    control_type = 'P'          # 控制类型: P, PD, torque
    stiffness = {              # PD 控制器刚度
        "joint": 20.0
    }
    damping = {                # PD 控制器阻尼
        "joint": 0.5
    }
    action_scale = 0.25        # 动作缩放系数
    decimation = 4              # 控制频率 = 仿真频率 / decimation
```

### rewards - 奖励设置

奖励配置分为两层：

1. **顶层参数**：奖励计算的通用参数
2. **scales**：各项奖励的权重

```python
class rewards:
    soft_dof_pos_limit = 0.9    # 关节软限制比例
    base_height_target = 0.36   # 目标基座高度
    only_positive_rewards = True # 是否只使用正奖励

    class scales:
        # 命令跟踪
        tracking_lin_vel = 1.0
        tracking_ang_vel = 0.5

        # 平滑性
        dof_vel = -5.e-4
        action_rate = -0.01

        # 限制
        dof_pos_limits = -1.0
        collision = -1.0

        # 步态
        feet_air_time = 1.0
        foot_clearance = 0.5
```

### commands - 命令生成

```python
class commands:
    curriculum = True           # 命令课程学习
    num_commands = 4            # 命令维度

    # 命令范围
    class ranges:
        lin_vel_x = [-0.5, 0.5]   # 前进速度 [m/s]
        lin_vel_y = [-1.0, 1.0]   # 侧向速度 [m/s]
        ang_vel_yaw = [-1.0, 1.0]  # 转向速度 [rad/s]
        heading = [-3.14, 3.14]    # 朝向角度 [rad]
```

### domain_rand - 领域随机化

```python
class domain_rand:
    randomize_friction = True
    friction_range = [0.5, 1.25]

    randomize_base_mass = True
    added_mass_range = [-1.0, 1.0]

    push_robots = True
    push_interval_s = 15
    max_push_vel_xy = 1.0

    randomize_com_displacement = True
    com_pos_range = [-0.01, 0.01]
```

## 4.4 训练配置详解

### 主要配置项

```python
class LeggedRobotCfgPPO:
    class algorithm:
        # PPO 算法参数
        value_loss_coef = 1.0
        use_clipped_value_loss = True
        clip_param = 0.2
        entropy_coef = 0.01
        num_learning_epochs = 5
        num_mini_batches = 4
        learning_rate = 1.e-3
        schedule = 'adaptive'  # 学习率调度

    class runner:
        policy_class_name = 'ActorCritic'
        runner_class_name = 'OnPolicyRunner'
        num_steps_per_env = 24          # 每个环境每轮收集的步数
        max_iterations = 1500           # 最大迭代次数
        save_interval = 200              # 保存间隔
        run_name = ''
        experiment_name = 'legged_robot'
```

## 4.5 配置继承关系

### 配置继承链

```
BaseConfig
  └─> LeggedRobotCfg (基础足式机器人配置)
        ├─> Go2FlatCommonCfg (Go2 通用配置)
        │     └─> GO2Cfg (Go2 基础任务配置)
        │           └─> GO2TSCfg (Go2 师生任务配置)
        └─> G1Cfg (G1 机器人配置)
```

### 通用配置：Go2FlatCommonCfg

为了避免重复代码，Go2 相关任务共享一个通用配置：

```python
class Go2FlatCommonCfg:
    class terrain:
        mesh_type = 'plane'
        # ...

    class init_state:
        pos = [0.0, 0.0, 0.36]
        # ...

    class control:
        control_type = 'P'
        # ...

    class asset:
        file = '{LEGGED_GYM_ROOT_DIR}/resources/robots/go2/urdf/go2.urdf'
        # ...
```

## 4.6 运行时参数覆盖

### 命令行参数

可以通过命令行参数覆盖配置：

```bash
python legged_gym/scripts/train.py --task go2 \
    --num_envs 1024 \
    --max_iterations 500 \
    --headless
```

### 常用命令行参数

| 参数 | 说明 |
|-----|------|
| `--task` | 任务名称 |
| `--num_envs` | 环境数量 |
| `--max_iterations` | 最大迭代次数 |
| `--headless` | 无渲染模式 |
| `--resume` | 恢复训练 |
| `--load_run` | 要加载的运行名称 |
| `--checkpoint` | 检查点迭代次数 |

## 4.7 配置最佳实践

### 1. 使用继承减少重复

利用配置继承，只覆盖需要修改的参数。

### 2. 分离环境和训练配置

环境配置定义任务，训练配置定义如何解决任务。

### 3. 使用合理的默认值

在基础配置中提供合理的默认值，减少子类配置的复杂度。

### 4. 注释关键参数

为非显而易见的参数添加注释说明。
