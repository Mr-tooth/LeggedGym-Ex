# 教程三：项目结构详解

## 3.1 整体架构

LeggedGym-Ex 采用模块化设计，代码分为三个核心部分：

```
LeggedGym-Ex/
├── legged_gym/     # 环境和模拟器接口
├── rsl_rl/         # 强化学习算法
└── resources/      # 机器人资产和数据
```

这种分离设计使得：
- 环境和算法解耦，可以独立开发
- 便于替换不同的模拟器
- 支持多种机器人平台

模拟器抽象基类，定义了所有模拟器必须实现的接口。

## 3.5 数据流和执行流程

### 训练流程

```
train.py
  ├─> 初始化模拟器
  ├─> task_registry.make_env()
  │   └─> 创建 LeggedRobot 环境
  │       └─> 创建 Simulator 实例
  ├─> task_registry.make_alg_runner()
  │   └─> 创建 OnPolicyRunner
  └─> runner.learn()
      └─> 循环：
          ├─> 收集经验（rollout）
          ├─> 更新策略
          └─> 保存模型
```

### 环境步进流程

```
env.step(actions)
  ├─> 预处理动作
  ├─> simulator.step(actions)  # 物理模拟
  └─> post_physics_step()
      ├─> check_termination()
      ├─> compute_reward()
      ├─> reset_idx(env_ids)
      └─> compute_observations()
```

## 3.6 关键设计模式

### 1. 策略模式（Strategy Pattern）

不同的模拟器实现相同的接口，可以互换使用。

### 2. 模板方法模式（Template Method Pattern）

`LeggedRobot` 定义了环境的整体流程，子类可以覆盖特定步骤。

### 3. 注册模式（Registry Pattern）

`TaskRegistry` 管理所有可用的任务，便于动态创建。
