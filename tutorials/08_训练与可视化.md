# 教程八：训练与可视化

## 8.1 训练流程详解

### 8.1.1 训练脚本结构

`train.py` 的完整流程：

```python
def train(args):
    # 1. 初始化模拟器
    if SIMULATOR == "genesis":
        gs.init(backend=gs.gpu, logging_level='warning')

    # 2. 创建环境
    env, env_cfg = task_registry.make_env(name=args.task, args=args)

    # 3. 创建算法运行器
    ppo_runner, train_cfg = task_registry.make_alg_runner(
        env=env, name=args.task, args=args)

    # 4. 备份配置文件
    log_dir = ppo_runner.log_dir
    shutil.copy(robot_file_path, log_dir)
    shutil.copy(robot_config_path, log_dir)

    # 5. 开始训练
    ppo_runner.learn(
        num_learning_iterations=train_cfg.runner.max_iterations,
        init_at_random_ep_len=True
    )
```

### 8.1.2 训练配置参数

```python
class LeggedRobotCfgPPO:
    class algorithm:
        value_loss_coef = 1.0           # 值函数损失权重
        use_clipped_value_loss = True    # 使用裁剪值函数损失
        clip_param = 0.2                  # PPO 裁剪参数
        entropy_coef = 0.01               # 熵正则化权重
        num_learning_epochs = 5           # 每轮迭代的更新次数
        num_mini_batches = 4              # 小批量数量
        learning_rate = 1.e-3             # 学习率
        schedule = 'adaptive'              # 学习率调度

    class runner:
        policy_class_name = 'ActorCritic'
        runner_class_name = 'OnPolicyRunner'
        num_steps_per_env = 24            # 每个环境每轮步数
        max_iterations = 1500              # 最大迭代次数
        save_interval = 200                # 保存间隔
        run_name = ''
        experiment_name = 'legged_robot'
```

### 8.1.3 训练超参数调优

| 参数 | 作用 | 调优建议 |
|-----|------|---------|
| `learning_rate` | 学习率 | 如果训练不稳定，降低到 5e-4；如果收敛慢，提高到 2e-3 |
| `clip_param` | PPO 裁剪 | 0.2 是标准值，特殊情况可调整到 0.1-0.3 |
| `entropy_coef` | 熵正则化 | 如果策略过早收敛，增加到 0.02-0.05 |
| `num_learning_epochs` | 更新次数 | 3-10 都可以，更多更新 = 更稳定但更慢 |
| `num_steps_per_env` | 每轮步数 | 24-48，确保覆盖完整的步态周期 |
| `num_mini_batches` | 小批量数 | 4-8，根据显存调整 |

## 8.2 训练监控

### 8.2.1 控制台输出

训练时会显示：

```
-------------------------------------------------------------------------------------------
|   iter    |  time / iter |  len / iter |   mean EPL   |   mean R   |   std R    |
-------------------------------------------------------------------------------------------
|    10     |    1.23s     |   122880    |    25.6      |    12.3    |    4.5     |
|    20     |    1.25s     |   122880    |    45.2      |    25.6    |    6.7     |
...
```

各列含义：
- `iter`: 迭代次数
- `time / iter`: 每次迭代耗时
- `len / iter`: 每次迭代的总步数
- `mean EPL`: 平均回合长度
- `mean R`: 平均奖励
- `std R`: 奖励标准差

### 8.2.2 TensorBoard 可视化

启动 TensorBoard：

```bash
tensorboard --logdir logs/go2
```

然后在浏览器打开 `http://localhost:6006`。

**关键指标**：
- `Reward/mean_episode_reward`: 平均回合奖励
- `Reward/mean_episode_length`: 平均回合长度
- `Policy/entropy`: 策略熵
- `Loss/policy_loss`: 策略损失
- `Loss/value_loss`: 值函数损失

## 8.4 播放脚本内部流程

### 8.4.1 播放脚本结构

```python
def play(args):
    # 1. 初始化模拟器
    if SIMULATOR == "genesis":
        gs.init(backend=gs.gpu, logging_level='warning')

    # 2. 加载配置并覆盖
    env_cfg, train_cfg = task_registry.get_cfgs(name=args.task)
    override_configs(env_cfg, args)

    # 3. 创建环境
    env, _ = task_registry.make_env(name=args.task, args=args, env_cfg=env_cfg)

    # 4. 加载策略
    train_cfg.runner.resume = True
    ppo_runner, train_cfg = task_registry.make_alg_runner(
        env=env, name=args.task, args=args, train_cfg=train_cfg)
    policy = ppo_runner.get_inference_policy(device=env.device)

    # 5. 导出模型
    export_policy(ppo_runner, path, args, env_cfg, train_cfg)

    # 6. 进入交互循环
    interaction_loop(env, policy, args)
```

### 8.4.2 配置覆盖

播放时会自动覆盖一些配置：

```python
def override_configs(env_cfg, args):
    # 减少环境数量
    env_cfg.env.num_envs = min(env_cfg.env.num_envs, 16)

    # 地形相关调整
    if env_cfg.terrain.mesh_type in ["heightfield", "trimesh"]:
        env_cfg.terrain.num_rows = 2
        env_cfg.terrain.num_cols = 2
        env_cfg.terrain.curriculum = False

        # 可以选择特定地形类型
        env_cfg.terrain.terrain_kwargs = {
            "type": "terrain_utils.pyramid_stairs_terrain",
            "step_width": 0.31,
            "step_height": -0.1,
            "platform_size": 3.0
        }

    # 启用调试模式
    env_cfg.env.debug = True

    # 游戏手柄时关闭朝向命令
    if args.use_joystick:
        env_cfg.commands.heading_command = False
```

## 8.5 模型导出和部署

### 8.5.1 导出模型

播放时会自动导出模型：

```bash
python legged_gym/scripts/play.py --task go2 --load_run <run_name> --export_onnx
```

导出的文件位置：
```
logs/go2/<run_name>/exported/
├── policy.jit    # PyTorch JIT 模型
└── policy.onnx   # ONNX 模型（如果 --export_onnx）
```

### 8.5.2 不同算法的导出

不同的任务类型使用不同的导出器：

| 任务类型 | 导出器 |
|---------|-------|
| 标准 PPO | `PolicyExporter` |
| TS/CaT | `PolicyExporterTS` |
| EE | `PolicyExporterEE` |
| DreamWaQ | `PolicyExporterWaQ` |

### 8.5.3 在 Python 中加载 JIT 模型

```python
import torch

# 加载模型
policy = torch.jit.load("policy.jit")
policy.eval()

# 推理
obs = torch.randn(1, 45)  # 假设观测维度是 45
with torch.no_grad():
    actions = policy(obs)

print(actions)
```

### 8.5.4 在 C++ 中加载 JIT 模型

```cpp
#include <torch/script.h>

int main() {
    // 加载模型
    torch::jit::script::Module module;
    module = torch::jit::load("policy.jit");

    // 准备输入
    std::vector<torch::jit::IValue> inputs;
    inputs.push_back(torch::randn({1, 45}));

    // 推理
    at::Tensor output = module.forward(inputs).toTensor();

    return 0;
}
```

## 8.7 常见问题

### Q: 训练时奖励不上升怎么办？

A:
1. 检查奖励配置是否合理
2. 确保初始姿势正确
3. 尝试降低学习率
4. 增加熵系数鼓励探索
5. 检查观测是否正确

### Q: 播放时机器人立即倒下？

A:
1. 确保加载了正确的检查点
2. 检查训练时和播放时的配置是否一致
3. 尝试加载更早的检查点（可能后期过拟合）

### Q: 如何在不同地形上测试？

A:
修改 `play.py` 中的 `override_configs` 函数，选择不同的地形类型。

### Q: 如何加速训练？

A:
1. 使用 IsaacGym 或 Genesis（不是 IsaacSim）
2. 增加 `num_envs`（只要显存够）
3. 减小 `save_interval`
4. 使用 `--headless` 模式
